# Reflection

## What I Learned

This activity helped me understand the basic Git and GitHub workflow. I learned how to fork a repository, clone it to my local machine, create and organize files, stage changes, commit them with meaningful messages, push changes to my GitHub repository, and finally create a Pull Request.

## Challenges Faced

Initially, I faced issues because Git was not installed correctly on my system. After installing and configuring Git, I was able to complete the workflow successfully. I also learned the importance of checking the repository status before committing changes.

## Key Takeaways

* Version control is essential for collaborative software development.
* Commits should be small and have meaningful messages.
* Pull Requests make collaboration easier by allowing code reviews before merging.
* GitHub provides an effective platform for managing projects and contributing to open-source repositories.

## Future Improvements

I plan to practice more Git commands such as branching, merging, resolving merge conflicts, and working with remote repositories to improve my collaboration skills.
